## Design Context

### Users

Power agents: developers and operators driving Pi / Addy auto workflows. They watch the dashboard while long-running auto loops execute across build, verify, review, and fix phases. They need at-a-glance status, accurate per-slice timings, and trustworthy progress data while moving between terminal and browser, often in split view or on a second monitor.

### Brand Personality

Operational, precise, momentum-driven. Three words: **focused, dependable, kinetic**. Voice is short and declarative, with no marketing copy. The UI should feel like a flight console for an autonomous coding agent: calm under load, but visibly moving when work is moving.

### Aesthetic Direction

Professional console with momentum cues.

- Dark OKLCH canvas using tinted neutrals around 0.14-0.22 lightness and hue near 245.
- Single restrained accent around `oklch(0.64 0.09 210)` plus semantic success, warning, and danger colors.
- Dense layout, strong hierarchy, tabular numerics, and compact controls.
- Subtle motion for state changes: progress bars, active/running indicators, and visible refresh cadence.
- Anti-references: glassmorphism, decorative gradients, hero metrics, em dashes, marketing illustrations, playful microcopy.

### Design Principles

1. **Truth over decoration.** Every visible value must map directly to workflow state; no fabricated summaries or unsupported aggregations.
2. **One thing per page.** Show one plan at a time; archive views are explicit through the plan dropdown, not mixed into the active view.
3. **Current state expanded, history collapsed.** The active slice/task opens by default; completed slices collapse to accumulated totals.
4. **Tabular and scannable.** Durations, counts, percentages, and task IDs align with tabular numerics for fast scanning.
5. **Restrained palette, semantic color only.** Color encodes state: active, complete, warning, failure. It is not decoration.
6. **Responsive but dense.** The UI adapts to tablet and mobile without hiding operational information; dense tables can scroll horizontally.
7. **Auto-refresh is a feature.** The 5 second cadence is visible so users can trust freshness.
