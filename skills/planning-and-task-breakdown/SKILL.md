# Planning and Task Breakdown

Plan read-only:

1. Read spec and relevant code.
2. Slice vertical user-visible increments.
3. Add acceptance criteria and verification for each task.
4. Add per-task status checkboxes for `[ ] Implemented`, `[ ] Verified`, and `[ ] Reviewed` so later workflow steps can keep the plan in sync with reality.
5. Mark dependencies and risky checkpoints.
6. Keep plan concise.

Do not implement until user explicitly orders implementation.
