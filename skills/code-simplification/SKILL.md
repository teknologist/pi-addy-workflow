# Code Simplification

Simplify only after understanding behavior and tests.

Look for:

- unnecessary nesting
- duplicated logic
- over-broad abstractions
- dead code
- unclear names
- long functions with separable responsibilities

Verify behavior before and after. Revert any simplification that changes behavior.
