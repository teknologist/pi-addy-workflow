# Spec-Driven Development

Before code:

1. Clarify objective, users, constraints, acceptance criteria.
2. Identify non-goals and risky assumptions.
3. Define exact verification commands or checks.
4. Save durable spec only when useful or requested.
5. Get user approval before implementation.

Keep specs concise and agent-verifiable.
