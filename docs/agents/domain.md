# Domain docs

This is a single-context repository.

Before exploring code:

- Read root `CONTEXT.md`.
- Read relevant decisions under `docs/adr/`.
- If either is absent, proceed silently.

Use vocabulary defined in `CONTEXT.md` in issues, plans, tests, and architecture
work. Avoid synonyms that the glossary explicitly rejects.

Surface conflicts with an ADR explicitly rather than silently overriding the
decision.
